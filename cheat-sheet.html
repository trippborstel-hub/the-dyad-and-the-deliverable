Cheat Sheet

The Dyad and the Deliverable

Key ideas, one glance
Companion to the essay synthesizing Diego Espinosa's Kith Climate lectures (with Ben Hillier). Eight sections, eight core moves. Full essay: trippborstel-hub.github.io/the-dyad-and-the-deliverable

01First, get in the car

The car tells you how to drive the car. You tell it where to go.
  • Tolerance for ambiguity is the entry-level mindset — nothing in AI is ironed out; treat the wobble as terrain.
  • Learning by raising the stakes: a real deliverable, due Friday, before you know how. "I don't know how to do this" is the point.
  • A purpose unlocks critical thinking. Deliverable-driven dialogue teaches collaboration; aimless chat teaches nothing.
  • Every mechanical question ("how do I…") → ask the tool. Destination, judgment, and worth → your department.
  • Empowerment test: the clerical work moves to the machine, the steering moves to you.

02The atomic unit

AI is not a model. It's a system by which humans get things done.
  • Four components: global inference (the model) + encoded local context (your data/methods) + computational environment (permissioned hands) + real-time human intelligence (you). Missing one = fooling around.
  • The model knows everything except what you know. Local context is the scarce input.
  • Why enterprises stall: smarter engines, still on the workbench — they bought inference and skipped the other three.
  • Prompt engineering is flat. The leverage isn't the incantation; it's context + judgment + veto power. Good prompting = asking a very smart person an open-ended question.

03Your IP lives in markdown

The firm's intellectual property is a folder of .md files.
  • CLAUDE.md = standing orders, read every session. Reference files (1–3 pages, one per methodology) = encoded institutional knowledge.
  • Never hand-write them; generate through iteration. Judge by output, not by rereading.
  • Reference ↔ inference spectrum: too little encoded → variance; too much → rigidity that strangles the intelligence. Find the middle empirically.
  • Pre-packaged skills = reference files with training wheels. The value is in the encoding experience, which can't be transferred.
  • Compounding: every deliverable becomes data for the next inference. Deliverables are a balance sheet, not clutter.
  • Determinism spectrum for facts: model memory → cited web search → markdown/SQLite → database + AI-written retrieval code. Move right as stakes rise.

04The number and the meaning

A number without financial context is a scoreboard. Connected to dollars, it's a finding.
  • Comprehensiveness matters when stakeholders hold you to X→Y. Most value lives between X and Y: risk, premiums, multiples, covenants.
  • Carbon is a placeholder for a process — the inventory is step one of a risk exercise (TCFD's actual origin).
  • The trust ladder: (1) model predicts the number → (2) model writes & runs code → (3) code you own. Guessed and computed look identical. Only rung 3 survives an auditor.
  • Recalibrate your clock. When execution collapses to minutes, the residual question is "what is the work, then?" Answer: partner judgment — what, why, for whom, worth what.

05Brilliant and stupid in the same conversation

Manage a colleague who is inconsistent at genius level.
  • Don't over-specify. Start broad, then narrow — micromanaged brilliance returns "but that's what you asked for."
  • Say less when correcting ("I noticed the deck was incomplete") and let it self-diagnose; then encode the fix in a reference file. Its self-corrections are brilliant — interns' aren't.
  • Keep a long-lived thinking session; spin off disposable doing sessions. Archive ruthlessly.
  • AI indigestion: volume tempts you to wave work through. Brief up a level — "plain English, one paragraph, the three things Sarah needs."
  • Polish seduces. Beautiful output suppresses questioning; a wall of charts can be the opposite of rigor.
  • Ask "how does it know that?" Pattern-stitching and fabrication wear the same confidence. Entertain both, every time.

06It's not software

Fallible like a person, tireless like a machine — a third kind of thing.
  • Forcing AI to behave like software is 2025 adoption — a category error. Software is bug-fixed and certain; this is neither.
  • The harness (memory + tools + autonomy) is currently worth more than the model. Test: is it workflow-complete — data in → reasoning → files out, loop closed?
  • Harness vs. app: the harness is where experts work (open-ended, judgment-scoped); the app is what you leave behind (codified inputs/outputs anyone can run). "I don't trust AI — build me an app" → okay, AI will build the app.
  • Agents are Kleenex — a paragraph of instructions plus plumbing, disposable. Write reference files; spawn agents as needed.
  • Use Claude to make Excel, not Claude in Excel. Whole aisles of the software store are training wheels now.
  • Adoption diagnostic: have you ever been rate-limited? If not, try harder. Change spreads one demo at a time, from below.

07Make every figure testable

Assurance never certified exactness — it certifies method, reasonableness, documentation.
  • Estimation was always the process. AI doesn't contaminate a precise craft; the benchmark is documentation, not perfection.
  • Random error washes out and spot-checks catch it. Systematic bias is the real threat: directional, recurring, accumulating.
  • Three bias loci: source data (not an AI problem) · orchestration (real but readable & fixable) · the weights (the honest carve-out — and exactly where a good system concentrates tagging + human review).
  • "Agent B checks Agent A" is not independence — same priors. Independence = external referents: stored sources, versioned factor APIs, deterministic calcs.
  • Frame every QA pass as find errors, never "confirm correctness."
  • Provenance is a binding, not an assertion — a checkable link to a stored source. Fabricated values have nothing to bind to: hallucination becomes a failed test, not a philosophy problem.
  • Pricing is the forcing function. "Good enough to charge X for" makes quality real. Work doesn't disappear — it gets reorganized and repriced.

08The dyad

The system is not the AI. The system is both of you together.
  • AI alone is a tree falling in the forest — value exists only through human judgment. One determinant of value in your context: you.
  • The model is the residual of our recorded patterns; it has no skin in the game — no stake, no pain, no "mattering." That's the stable human seat.
  • Users know things the builders don't. Intensive operators of the dyad hold original knowledge — contribute it.
  • Variance is the evidence: same assignment, same model, 28 different deliverables. The more difference, the more the human matters.
  • The scarcest resource in AI is human imagination. Use it freely.
  • Teacher's endgame: raise the stakes, hand over the keys, make yourself progressively unnecessary.
